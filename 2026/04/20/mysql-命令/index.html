MySQL 学习笔记

一、数据库基础操作

sql

– 查询数据库
show databases;
– 查询当前数据库
select database();
– 创建数据库
create database 数据库名 [if not exists];
– 删除数据库
drop database 数据库名;
– 使用数据库
use 数据库名;  

二、数据表操作

  1. 表查看与创建

sql

– 查询库中所有表
show tables;
– 查看表结构
desc 表名;
– 查询指定表的建表语句
show create table 表名;

– 创建表
create table 表名(
字段1 字段类型 [约束],
字段2 字段类型 [约束],
…
) [comment 表注释];

– 常用数据类型
int – 整数类型
varchar – 字符串类型

– 常用约束
非空约束: NOT NULL
唯一约束: 不允许出现重复 unique
主键约束: primary key (自增 auto_increment)
默认约束: default
检查约束: check
外键约束: foreign key  

  1. 表修改

sql

– 添加字段
alter table 表名 add 字段名 类型 [comment 注释] [约束];
– 修改字段名和字段类型
alter table 表名 change 旧字段名 新字段名 类型 [comment 注释] [约束];
– 修改数据类型
alter table 表名 modify 字段名 新数据类型;
– 删除字段
alter table 表名 drop 字段名;
– 修改表名
alter table 表名 rename to 新表名;
– 删除表
drop table [if exists] 表名;  

三、数据操作（增删改查）

sql

– 添加数据
insert into 表名 (字段1, 字段2, …) values (值1, 值2, …)[, (值1, 值2, …)];
– 修改数据
update 表名 set 字段1=值1, 字段2=值2, … [where 条件];
– 删除数据
delete from 表名 [where 条件];  

四、查询语法与多表操作

  1. 基础查询语法

sql

select 字段列表 – 字段名(AS)别名
from 表名
where 条件列表 – >、>=、<、<=、<>、like、between…and、in、and、or
group by 分组字段列表 – 分组前过滤（where）
having 分组后条件列表 – 分组后过滤（having）
order by 排序字段列表 – 升序ASC、降序DESC
limit 分页参数 – 起始索引(从0开始), 每页记录数;  

  1. 多表查询

sql

– 隐式内连接
select * from 表A, 表B where 条件;
– 显式内连接
select … from 表A inner join 表B on 条件;
– 左外连接
select … from 表A left join 表B on 条件;
– 右外连接
select … from 表A right join 表B on 条件;
– 自连接
select … from 表A 别名1, 表A 别名2 where 条件;  

  1. 子查询

sql

– 标量子查询、列子查询、行子查询、表子查询
– 列子查询常用操作符
in – 在指定的集合范围内，多选一
not in – 不在指定的集合范围内
any – 子查询返回列表中，有任一满足即可
some – 与any等同
all – 子查询返回列表所有值必须满足  

五、常用函数

  1. 字符串函数

sql

concat – 拼接字符串
lower – 全转为小写
upper – 全转为大写
lpad – 左填充
rpad – 右填充
trim – 去除左右两边空格
substring – 字符串截取  

  1. 数值函数

sql

ceil – 向上取整
floor – 向下取整
mod – 求模
rand – 求随机数(0-1)
round – 四舍五入保留指定位数  

  1. 日期函数

sql

curdate – 获取当前日期
curtime – 获取当前时间
now – 当前日期+时间
year – 获取年份
month – 获取月份
day – 获取日期
date_add – 添加指定时间周期
datediff – 两个日期相减  

  1. 流程函数

sql

if(value, t, f) – 如果value为true，返回t，否则返回f
ifnull(value1, value2) – 如果value1不为空，返回value1，否则返回value2
case when [条件1] then [结果1] … else [默认结果] end – 如果条件1满足，则返回结果1，否则执行下一个条件
case expr when [值1] then [结果1] … else [默认结果] end – 如果expr等于值1，则返回结果1，依此类推  

六、用户与权限管理

sql

– 创建用户
create user ‘用户名‘@’主机名’ identified by ‘密码’;
– 修改用户密码
alter user ‘用户名‘@’主机名’ identified with mysql_native_password by ‘新密码’;
– 删除用户
drop user ‘用户名‘@’主机名’;

– 权限控制
grant 权限列表 on 数据库名.表名 to ‘用户名‘@’主机名’;
revoke 权限列表 on 数据库名.表名 from ‘用户名‘@’主机名’;  

七、事务管理

  1. 事务提交与回滚

sql

– 查看事务提交方式
select @@autocommit;
– 设置事务提交方式
set @@autocommit = 0;
– 开启事务
start transaction 或 begin;
– 提交事务
commit;
– 回滚事务
rollback;  

  1. 事务四大特性（ACID）
  • 原子性：事务是不可分割的最小操作单位，要全部成功或全部失败
  • 一致性：事务完成时，必须使所有数据一致
  • 隔离性：数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
  • 持久性：事务一旦提交或回滚，它对数据库的改变是永久的
  1. 并发事务问题
  • 脏读：一个事务读到另一个事务还没有提交的数据
  • 不可重复读：一个事务先后读取同一条记录，但两次读取的数据不同
  • 幻读：一个事务按照条件查询数据时，没有对应的数据行，但在插入数据时，又发现这行数据已经存在
  1. 事务隔离级别

隔离级别 脏读 不可重复读 幻读 性能
Read Uncommitted 高
Read Committed 中
Repeatable Read（MySQL默认） 中
Serializable 低

sql

– 查看事务隔离级别
select @@transaction_isolation;
– 设置事务隔离级别
set [session/global] transaction isolation level [Read Uncommitted/Read Committed/Repeatable Read/Serializable];
